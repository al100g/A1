# Contributing to A1

Thank you for your interest in contributing to the A1 project! We are excited to have you on board. Below are the guidelines for getting started and contributing effectively:

## Setting Up Development Environment
1. **Clone the repository**:
   ```bash
   git clone https://github.com/al100g/A1.git
   cd A1
   ```

2. **Install dependencies**:
   Make sure you have all required dependencies. Run the following command:
   ```bash
   npm install
   ```

3. **Run the application**:
   You can start the development server by running:
   ```bash
   npm run dev
   ```

## Making Changes
- Create a new branch for your feature or bug fix:
  ```bash
  git checkout -b feature/your-feature-name
  ```

- Make your changes and be sure to test them!
- You can check the status of your changes with:
  ```bash
  git status
  ```

## Submitting Pull Requests
1. **Commit your changes**:
   ```bash
   git add .
   git commit -m "Description of your changes"
   ```

2. **Push your branch**:
   ```bash
   git push origin feature/your-feature-name
   ```

3. **Create a Pull Request**:
   - Go to the GitHub repository and click on "Pull Requests".
   - Click on "New Pull Request".
   - Select your branch and create the pull request.

4. **Fill out the PR template** (if applicable) and submit it.

## Code Standards
- Follow the existing code style and conventions used in the project.
- Write clear, descriptive commit messages.
- Include comments in your code where necessary to explain complex logic.

Thank you for contributing! We appreciate your help in making A1 better for everyone.
